## CHAPTER 7 — Job Completion & Account Compliance

### 7.1 Expected Completion Date — Your Job Deadline
Added April 8, 2026.

When a Scope of Work is created and approved, an **Expected Completion Date** is set. Both contractor and client must close out the job (confirm done, after-photos uploaded, payment processed) by this date.

**What Happens When Overdue:**
- Both contractor and client accounts are locked for non-platform activities.
- Locked accounts can still access direct messaging with each other.
- All other platform features blocked until the overdue job is closed out.

**How to Resolve:** Complete job closeout → scope status changes to **Closed** → both accounts automatically and instantly unlocked. Confirmation email sent.

### 7.2 Account Lock System — Types & How It Works
- **After-Photo Lock:** Triggered 72 hours after agreed work date if after-photos missing. Lifts immediately when photos uploaded.
- **Rating Block:** Triggered when completed job hasn't received mutual rating. Only non-compliant party is blocked.
- **Overdue Job Lock:** Triggered when job passes expected completion date without closeout. Locks both accounts.

**What You Can Still Do While Locked:** Message the other party, access existing job documentation, upload after-photos or submit ratings.

**What Is Blocked:** Accepting new jobs, posting new listings, making new payments, non-essential platform features.

All locks are fully automated — no manual review process.

### 7.3 Billing Cycle & Account Deletion
⚠️ Only applies to accounts that remain locked through end of billing cycle without resolution.

If contractor's account remains locked for an overdue job and billing cycle ends without resolution:
1. Payment card details removed from account.
2. Account queued for permanent deletion.
3. Both contractor and admin notified by email.

**How to Avoid:** Always close out jobs on time. If more time needed, update the scope's expected completion date *before* deadline passes. Never accept payment outside SurfCoast.

### 7.4 One Active Job at a Time
Once a Scope of Work is approved, contractor is marked **On Active Job** — prevents accepting additional scopes until current job closes out.

**What Changes When Job Closes:** On Active Job flag cleared, availability resets to Available, can immediately begin accepting new scopes.

## CHAPTER 31 — Platform Security & Backend Architecture — April 23, 2026

### 31.1 Authentication & Authorization Model

**Authentication Provider:** Base44 platform authentication system. All login, session management, token issuance, and email verification handled at the platform level. Tokens automatically attached to all SDK requests.

**Three Access Levels:**
- **User (default):** Authenticated platform user — can read/write only their own data per RLS rules.
- **Admin:** Elevated role — bypasses most RLS restrictions. Required for all admin dashboards and enforcement functions.
- **Service Role:** Internal backend identity — used by automated functions running without a user session. Never exposed to the frontend.

**INTERNAL_SERVICE_KEY:** Scheduled and internal-only backend functions are protected by `INTERNAL_SERVICE_KEY` secret. Callers must include this in the `x-internal-key` request header. Functions without user sessions check for either the platform automation header (`x-automation-id`) or this key.

### 31.2 Row-Level Security (RLS) — Entity Access Control

**What RLS Does:** Every entity has Row-Level Security rules controlling who can create, read, update, or delete each record. Enforced at the database query level — cannot be bypassed with frontend tricks or API calls.

**Key RLS Patterns Used:**
- **Email-scoped:** `data.poster_email === user.email` — only record owner can access.
- **Dual-party access:** ScopeOfWork readable by both `contractor_email` and `client_email`.
- **Public read with demo exclusion:** Jobs and Reviews use `$and [status: open, is_demo: false]` to show only real live records to anonymous users.
- **Admin override:** Every entity has `user_condition: role: admin` bypass.
- **Service role pass-through:** `base44.asServiceRole` bypasses RLS entirely — used only for trusted server-side operations.

**Demo Data Isolation (April 23, 2026):** All seed/demo records using `@demo.surfcoast.com` emails are tagged with `is_demo: true`. RLS read rule on Job and Review entities explicitly excludes `is_demo: true` records from public queries — test data never surfaces in the live site.

### 31.3 Backend Functions & Automation Security

**Function Types:**
- **User-invoked:** Called from frontend via SDK. Require `base44.auth.me()` authentication. Return 401 if unauthenticated.
- **Admin-only:** Require `user.role === 'admin'` after authenticating. Return 403 if not admin.
- **Entity automation:** Triggered by database events. No user session — validated via structural payload check: must contain `event.type + event.entity_name + event.entity_id`.
- **Scheduled automation:** Triggered by platform cron. Validated via `x-automation-id` header or `INTERNAL_SERVICE_KEY`.

**Security Fix — April 23, 2026:** 13 backend functions had a shared vulnerability: they used `!!body.event` to detect automation payloads and skip authentication — meaning any caller who sent a JSON body containing an `event` key could bypass access controls entirely.

All 13 functions were patched to require structural validation:

```javascript
// BEFORE (vulnerable)
const isAutomation = !!body.event;

// AFTER (secure)
const isValidAutomation =
  body?.event?.type &&
  body?.event?.entity_name &&
  body?.event?.entity_id;
```

**Functions Patched (April 23, 2026):**
onReviewCreated, onProfileCompleted, notifyNewProjectMessage, notifyFoundingMemberBenefit, scheduledLeaderboardUpdate, scheduledLowStockCheck, notionAutoSync, autoUnlockOnJobClose, generateInvoiceOnCloseout, syncContactToHubSpot, syncDealToHubSpot, processToolSuggestion, checkMinorHoursLimit.

### 31.4 Entity Automations — How They Work

**Automation Payload Structure:**
```json
{
  "event": {
    "type": "update",
    "entity_name": "ScopeOfWork",
    "entity_id": "abc123"
  },
  "data": { "...current entity fields..." },
  "old_data": { "...previous fields (update only)..." },
  "changed_fields": ["status", "after_photo_urls"]
}
```

**Active Automations on the Platform:**
- **onReviewCreated:** Recalculates contractor average rating and review count when new review created.
- **onProfileCompleted:** Fires referral rewards and extends trial days when contractor/customer profile completion is detected.
- **notifyNewProjectMessage:** Sends in-app notification + email when a ProjectMessage is created on an active scope.
- **notionAutoSync:** Creates/updates Notion project pages when ScopeOfWork is created or updated, or when milestone completed.
- **autoUnlockOnJobClose:** Unlocks both contractor and client accounts automatically when scope status changes to "closed."
- **generateInvoiceOnCloseout:** Generates PDF invoice and emails both parties when scope is closed out.
- **syncContactToHubSpot / syncDealToHubSpot:** Syncs contractor/customer profiles and job deals to HubSpot CRM on record changes.
- **processToolSuggestion:** AI-processes tool suggestions submitted during entrepreneur onboarding, updates aggregate mention count.
